# SkillRPG Skill Tracker

SkillRPG Skill Tracker — це веб-додаток для керування прогресом команди через систему задач, бійців та навичок. Програма допомагає координувати складні навчальні або онбордингові плани, відстежувати затвердження XP й швидко реагувати на блокери.

## Ключові можливості

- **Дошка задач** з колонками статусів (`todo`, `in_progress`, `validation`, `done`, `archived`).
- **Модальні вікна** для створення задач, перегляду деталей та масового призначення.
- **Трекінг навичок** бійців з обліком XP і коментарями.
- **Фільтрація та пошук** для фокусування на потрібних задачах або виконавцях.
- **Імпорт/експорт** даних (JSON/CSV) для резервного копіювання чи інтеграцій.

## Стек

- [React 18](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
- [Vite](https://vitejs.dev/) для збірки та дев-сервера
- [Vitest](https://vitest.dev/) + Testing Library для тестів
- [TypeDoc](https://typedoc.org/) для API-документації

## Передумови

- Node.js 18+ (рекомендовано LTS)
- npm 9+

## Швидкий старт

```bash
npm install
npm run dev
```

Дев-сервер стартує на `http://localhost:5173` (порт можна змінити через опцію `--port`).

## Скрипти npm

- `npm run dev` — запуск дев-сервера Vite.
- `npm run build` — збірка TypeScript та продакшн-бандла.
- `npm run preview` — превʼю продакшн-збірки.
- `npm test` — запуск усіх тестів у режимі одного прогону.
- `npm run coverage` — тестування з підрахунком покриття.
- `npm run docs:api` — генерація API-документації у `docs/api/` за TypeDoc.

## Тестування

Проєкт використовує Vitest і Testing Library. Рекомендації зі структурування тестів описані у [docs/testing-templates.md](docs/testing-templates.md). Перед створенням PR виконуйте `npm test` для перевірки стабільності.

## Документація

- [docs/radar.md](docs/radar.md) — огляд архітектурних конвенцій.
- [docs/ui-guidelines.md](docs/ui-guidelines.md) — UI-патерни та спільні компоненти.
- [docs/testing-templates.md](docs/testing-templates.md) — шаблони тестування.
- [docs/typedoc.md](docs/typedoc.md) — інструкції з генерації API-доків.

API-документація генерується командою `npm run docs:api` і зберігається в `docs/api/` (ігнорується Git за замовчуванням).

## Структура проєкту

```text
src/
  components/        # UI-компоненти, згруповані за доменами
  hooks/             # Реюзабельні логічні хуки стану
  lib/               # Утиліти та доменні допоміжні функції
  pages/             # Сторінки застосунку та їхні частини
  types.ts           # Доменні типи (бійці, навички, задачі тощо)
docs/                # Додаткова документація проєкту
vite.config.ts       # Налаштування Vite
vitest.config.ts     # Конфігурація тестового раннера
```

## Подальший розвиток

Ідеї майбутніх покращень (заплановані раніше):

1. Винести інлайн-стилі в CSS/утиліти.
2. Перенести дата-адаптери до `utils/` з тестами.
3. Впровадити покращене керування станом форм (наприклад, через власні хуки або `react-hook-form`).
4. Створити реюзабельний хук для керування модалками.
5. Строго типізувати імпорт/експорт JSON/CSV.
6. Додати компонент підтвердження для небезпечних дій.
7. Розширити інтеграційні тести (drag&drop, імпорт/експорт, модальні флоу).
8. Покращити тему через пресети та збереження налаштувань.
9. Розбити головну сторінку на менші типізовані компоненти.
10. Додати юніт-тести та локалізацію для CSV-експорту.

## Внесок

- Створюйте фічі у відповідних доменних директоріях (`src/components/<Domain>`).
- Додавайте README/документацію до нових великих модулів.
- Супроводжуйте складні зміни тестами.
- Перед PR виконуйте `npm run build` і `npm test`.
